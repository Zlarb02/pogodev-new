OPTIONS TECHNIQUES AVANCÉES

OPTION 1 - APIs & Automatisations (IA incluse) :
Intégration d'APIs — y compris LLM — si ça résout un problème réel.

Exemple concret : prototype d'application immersive augmentée par IA
(génération de contenu dynamique, expérience narrative type AI Dungeon).

⚠️ Transparence sur l'IA :
L'IA est un outil puissant, mais pas sans risques : biais algorithmiques,
hallucinations, consommation énergétique, dépendance aux modèles propriétaires.

Je l'utilise quand le bénéfice est clair (automatisation, gain de temps,
amélioration qualité du code). Pas pour le buzz marketing.

Bilan complexe : 1 dev + IA vs 10 devs en bureau avec déplacements quotidiens
→ quel impact réel ? Difficile à trancher. La transparence sur les choix et
leurs conséquences me semble plus importante que la peur ou l'enthousiasme
aveugle.

OPTION 2 - Stack sans GAFAM :
Hébergement perso/VPS, DNS maîtrisé, analytics légers open-source, pas de
tracking tiers.

Pourquoi ? Contrôle des données, souveraineté numérique, résilience.
C'est un choix technique selon vos priorités.

OPTION 3 - Déploiement dockerisé :
Stacks dockerisées déployables avec Traefik/Nginx. Hébergement simple,
reproductible, évolutif. Vous pouvez migrer facilement d'un hébergeur à l'autre.

OPTION 4 - Dashboards d'administration sur-mesure :
Interfaces React custom pour gérer votre contenu en autonomie. Pas de CMS
générique — un outil taillé pour votre workflow.

PAGE ÉCO-CONCEPTION

TITRE PRINCIPAL :
L'éco-conception web : pourquoi ça marche (et pourquoi c'est rentable)

INTRO :
Créer des sites performants, sobres et accessibles. Pas par militantisme,
mais parce que les chiffres parlent d'eux-mêmes : ça coûte moins cher,
ça charge plus vite, et ça convertit mieux.

Voici comment ça marche, avec des données vérifiables.

---

POURQUOI S'EN PRÉOCCUPER ?

CARTE 1 - Impact environnemental mesurable :
Le numérique représente 4% des émissions mondiales de CO₂ (source : The Shift
Project, 2019). C'est plus que l'aviation civile.

Chaque octet transféré consomme de l'énergie (serveur + réseau + terminal).
Moins il y en a, moins ça coûte — en euros et en CO₂.

CARTE 2 - Performance = argent :
Un site éco-conçu charge 2 à 3 fois plus vite (mesures Lighthouse).

Impact direct :
• +1s de temps de chargement = -7% de conversions (source : Akamai)
• Google pénalise les sites lents dans le ranking SEO
• Serveurs moins puissants = facture d'hébergement réduite

CARTE 3 - Accessibilité = inclusivité :
Sites légers = utilisables sur connexions faibles (3G, zones rurales) et
vieux appareils (smartphones 2016-2018).

En France, 13% de la population n'a pas accès au haut débit (ARCEP, 2023).
Un site de 5 Mo est inutilisable pour ces profils. Un site de 500 Ko, oui.

---

LES 6 PILIERS DE L'ÉCO-CONCEPTION

1. OPTIMISATION DES IMAGES
Les images = 50% du poids d'une page en moyenne (HTTPArchive, 2024).

• Formats modernes : WebP (-30%), AVIF (-50% vs JPEG)
• Compression : qualité 80-85% (perte imperceptible à l'œil)
• Lazy loading : charger seulement au scroll
• Dimensions adaptées : pas d'image 4K pour un thumbnail 300px

Résultat mesuré sur Alto Lille : -93% de poids images (3.4 Mo → 224 Ko AVIF).

---

2. CODE SOBRE ET MINIFIÉ
Éviter le code mort et les dépendances inutiles.

• Tree-shaking : supprimer le code non utilisé (-40% bundle JS moyen)
• Code-splitting : charger uniquement le nécessaire (import() dynamique)
• Audit régulier : bundlephobia.com pour mesurer le poids des packages npm

Exemple concret : une bibliothèque JS de 200 Ko pour afficher un bouton ?
Non merci. Un composant custom de 2 Ko fait le job.

Résultat mesuré : bundle JS initial de 872 Ko → 424 Ko après optimisation
(-51% via lazy-loading Three.js et suppression de dépendances inutiles).

---

3. TYPOGRAPHIE OPTIMISÉE
Les fonts Google peuvent peser 200-400 Ko par famille (2 familles = 600 Ko).

• Variable fonts : 1 seule font = tous les poids (400 à 700) → -60%
• Subsetting : charger uniquement les caractères latin (pas cyrillique/chinois)
• display=swap : afficher le texte immédiatement (pas de FOIT)
• Auto-hébergement : évite les requêtes externes Google Fonts (DNS + HTTP)

Résultat : Inter auto-hébergée = 344 Ko pour tous les poids vs ~500 Ko via Google.

---

4. HÉBERGEMENT RESPONSABLE
Le choix de l'hébergeur a un impact mesurable.

• Énergie verte : datacenters alimentés aux EnR (Infomaniak, OVH Eco, o2switch)
• Proximité : CDN ou hébergement proche des utilisateurs (latence + bande passante)
• Mutualisation : VPS partagé plutôt que serveur dédié si trafic < 10k/mois

Impact mesuré : un serveur dédié sous-utilisé (10% CPU) = gaspillage électrique.
Un VPS mutualisé = ressources partagées, meilleur rendement énergétique.

---

5. CORE WEB VITALS (métriques Google)
Mesures objectives de l'expérience utilisateur.

• LCP (Largest Contentful Paint) < 2.5s : affichage contenu principal
• FID (First Input Delay) < 100ms : réactivité aux interactions
• CLS (Cumulative Layout Shift) < 0.1 : stabilité visuelle (pas de décalage)

Ces métriques impactent directement le SEO depuis 2021 (Google Page Experience).

Résultat Alto Lille : 100/100 sur Lighthouse (Performance + Accessibilité + SEO).

---

6. ACCESSIBILITÉ & SOBRIÉTÉ
Un site accessible est un site sobre par construction.

• Sémantique HTML : balises natives (nav, main, article) → compréhensibles
  par tous (lecteurs d'écran, moteurs de recherche, anciens navigateurs)
• prefers-reduced-motion : respecter les préférences utilisateur (épilepsie,
  sensibilité au mouvement)
• Contraste : ratio WCAG AAA minimum (7:1) → lisible par malvoyants

Impact mesuré : 15% de la population mondiale a un handicap (ONU). Un site
inaccessible exclut potentiellement 15% de votre audience.

---

OUTILS D'AUDIT (gratuits et open-source)

• Lighthouse : audit de performance/accessibilité/SEO (Chrome DevTools)
• WebsiteCarbon : calculateur d'empreinte CO₂ par page
• EcoIndex : note environnementale A-G (collectif GreenIT)
• bundlephobia : analyse le poids des packages npm avant installation

Tous vérifiables, tous reproductibles.

---

ROI DE L'ÉCO-CONCEPTION

Investissement : optimisation initiale (~10-20h dev)

Retours mesurables :
• Hébergement : serveur 2x moins puissant → -50% facture mensuelle
• SEO : meilleur ranking Google → +trafic organique gratuit
• Conversion : -1s chargement → +7% conversions (Akamai)
• Maintenance : code simple → -temps debug → -coût évolution

Exemple chiffré (site e-commerce 10k visiteurs/mois) :
• Avant : serveur 40€/mois + taux rebond 60% + SEO page 3
• Après : serveur 20€/mois + taux rebond 45% + SEO page 1
→ ROI en 3-6 mois selon trafic.
