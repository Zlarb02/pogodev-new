TITRE PRINCIPAL :
L'éco-conception web, c'est quoi ?

INTRO :
Créer des sites performants, sobres et accessibles. Pas par militantisme,
mais parce que c'est logique : ça coûte moins cher, ça charge plus vite,
et ça marche mieux pour tout le monde.

Voici comment ça marche.

---

POURQUOI S'EN PRÉOCCUPER ?

CARTE 1 - Impact environnemental :
Le numérique, c'est 4% des émissions mondiales de CO2. Plus que l'aviation.
Chaque octet transféré consomme de l'énergie. Moins il y en a, mieux c'est.

Pas de culpabilisation. Juste des faits.

CARTE 2 - Performance :
Un site éco-conçu charge 2 à 3 fois plus vite. Meilleur pour le SEO,
meilleur pour l'expérience utilisateur. Win-win.

CARTE 3 - Accessibilité :
Sites légers = accessibles sur connexions faibles et vieux appareils.
Inclusivité numérique. Pas élitiste.

---

LES 6 PILIERS DE L'ÉCO-CONCEPTION

1. OPTIMISATION DES IMAGES
Les images = 50% du poids d'une page en moyenne.

• Formats modernes : WebP (-30%), AVIF (-50%), SVG pour icônes
• Compression : qualité 80-85%, dimensions adaptées
• Lazy loading : charger seulement au scroll

Résultat : -70% de poids sans perte visible.

---

2. CODE SOBRE ET MINIFIÉ
Éviter le code mort, supprimer les dépendances inutiles.

• Tree-shaking : supprimer le code non utilisé (-40% bundle)
• Code-splitting : charger uniquement ce qui est nécessaire (import() dynamique)
• Audit dépendances : bundlephobia.com pour analyser le poids

Une bibliothèque JS de 200 Ko qui sert à afficher un bouton ? Non merci.

---

3. TYPOGRAPHIE OPTIMISÉE
Les fonts Google peuvent peser 200-400 Ko par famille.

• Variable fonts : 1 seule font = tous les poids (-60%)
• Subsetting : charger uniquement les caractères utilisés
• display=swap : afficher le texte immédiatement

Ou auto-héberger pour éviter les requêtes externes.

---

4. HÉBERGEMENT RESPONSABLE
Le choix de l'hébergeur a un impact direct.

• Énergie verte : datacenters alimentés aux énergies renouvelables
• Proximité : hébergement proche des utilisateurs (CDN)
• Mutualisé : VPS partagé plutôt que serveur dédié si possible

Là encore, pas de moralisme. Juste du pragmatisme.

---

5. CORE WEB VITALS
Métriques Google pour mesurer l'expérience utilisateur.

• LCP (Largest Contentful Paint) < 2.5s
• FID (First Input Delay) < 100ms
• CLS (Cumulative Layout Shift) < 0.1

Ces métriques impactent le référencement. Autant les soigner.

---

6. ACCESSIBILITÉ & SOBRIÉTÉ
Un site accessible est un site sobre par nature.

• Sémantique HTML : utiliser les bonnes balises (nav, main, article)
• prefers-reduced-motion : respecter les préférences utilisateur
• Contraste : WCAG AAA minimum (ratio 7:1)

Accessible = utilisable par tous. Logique.

---

OUTILS D'AUDIT RECOMMANDÉS

• Lighthouse : audit de performance, accessibilité et SEO (Chrome DevTools)
• WebsiteCarbon : calculateur d'empreinte carbone
• EcoIndex : note environnementale de A à G (GreenIT.fr)
• bundlephobia : analyse le poids des packages npm
